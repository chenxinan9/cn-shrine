## 一、版本号怎么打?

### 1.1 命名规则

```
[神名]_[类型]-V[大版本].[小版本]_[日期].扩展名
```

例:

```
悟空_业务代码-V1.0_20260601.code
白泽_审计报告-V0.3_复查_20260601.md
白泽_审计报告-V0.4_终审_20260601.md
沙僧_DESIGN-V2.1_token升级_20260601.md
```

### 1.2 大版本 vs 小版本

- **大版本(V1.0 / V2.0)** = 大改 · 不兼容
- **小版本(V1.1 / V1.2)** = 小改 · 兼容上一版

### 1.3 定稿了怎么锁?

你在第 8 步(双审闭环最后)打个稳定标签:

```
定稿-vX.X-日期
```

锁定后:

- ✅ 任何改动 → 必须开新版本号
- ❌ 不覆盖 · 不偷偷改
- ❌ 不"我加个小补丁就行"

锁定后改:
1. 开新版本号(V1.1 / V2.0)
2. 走双审闭环 8 步全程
3. 你再打稳定标签

## 二、交接单(神跟神之间)

### 2.1 6 个必填字段

每次跨神交接,必须包含:

| 字段 | 写什么 |
|---|---|
| **从** | 谁交的 |
| **到** | 给谁 |
| **内容** | 真物路径 / 文件名 |
| **版本** | V0.X · 配命名规范 |
| **依赖** | 需要等谁的什么才能干完 |
| **占位符** | 哪些数字 / 内容待填 |
| **铁律提醒** | 下游神必守的禁区 |

详例:`examples/01-交接单模板.md`

### 2.2 干嘛要交接单?消灭"我以为你知道"

跨神协作最大坑 = 默认假设。

```
悟空:"我以为白泽会自己查红线"
白泽:"我以为悟空会先自查红线"
→ 都没查 · 红线漏审 · 你怒
```

交接单 = **把假设全写下来** · 没有"我以为"。

## 三、回滚预案

### 3.1 每个上线阶段必定义

```
触发回滚的情况:
  - 例:用户投诉率 > X%
  - 例:红线被碰
  - 例:串味报告 > 0

回滚操作:
  - 切回上一版稳定标签
  - 通知唐僧 + 白泽
  - 24h 内出事故报告

谁有权回滚: 你(唯一)
```

详例:`examples/02-回滚预案模板.md`

### 3.2 反"AI 偷偷回滚"

回滚是高风险 · 任 1 神不能擅自:

| ❌ 错 | ✅ 正 |
|---|---|
| 悟空发现 bug 偷偷 revert | 立即告诉你 · 候你拍 |
| 白泽审出严重问题立刻回 | 反第 5 条铁律(找茬的不修)· 必你 |
| 唐僧"我觉得这版有风险" | 跨身份建议 · 候你决 |

## 五、算账(成本与效率)

详 `15-成本与效率记录模板.md`。

每个阶段记录:

- 跑了多久(墙钟时间)
- AI 跑了多久(纯产出时间)
- 你投入了多久(转 prompt + 拍板)
- 最后出了多少真物
- 红线被碰几次
- 串味几条

将来分享时,这组数字最有说服力:
> "1 个人 + N 个 AI · 48 小时 · 出了 P 件东西 · 0 红线违规"

## 六、整体跑下来 11 步

```
1. 干活 → 版本号 V0.X
2. 自查 → 命名规范 + 真物
3. 独立查 → 交接单给白泽
4. 你拍 → 接受 / 反驳 / 部分接受
5. 修 → 新版本号 V0.X+1
6. 复查 → 交接单给白泽 · 复查报告
7. 定稿 → 打稳定标签
8. 记经验 → SKILL 卡片入库
9. 上线 → 触发条件预定义
10. 回滚 → 你唯一权(如果触发)
11. 算账 → 成本效率记录
```
