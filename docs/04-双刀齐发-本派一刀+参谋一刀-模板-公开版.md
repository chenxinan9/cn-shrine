# 双刀齐发 · 公开版模板

> 任何 AI 给大产出 · 必带"专业建议 + 战略 3 件套"双刀 · 防偷工 + 防"看似交付"

## 一、双刀定义

### 第 1 刀 · 本派一刀(专业建议)

**是**:对**当前具体产出**的专业建议(基于真物 · 不是自检表)
**不是**:堆"X/10 自检表"形式
**格式**:1-3 条 · 每条 ≤ 50 字 · **必带 利 ≥ 1 / 害 ≥ 1 / 守底线 ≥ 1**

### 第 2 刀 · 参谋一刀(战略 3 件套)

**A 改进建议** · 1-3 条 · 立即可干
**B 风险预警** · 1-3 条 · 标 🔴 极急 / 🟡 候议
**C 未发现机会** · 1-3 条 · 轻提(不放大 · 候明示再深耕)

## 二、模板(可复制)

```markdown
## 一刀(对当前 {对象} 的专业建议)

1. **利**:{具体守 / 强项 · ≤ 50 字 · 带真物证}
2. **害**:{具体破 / 弱点 · ≤ 50 字 · 带改进方向}
3. **守底线**:{法律 / 红线 / 反夸张 · ≤ 50 字}

## 参谋一刀

### A 改进建议
- A1 {具体动作 · 立即可干}
- A2 ...

### B 风险预警
1. 🔴 {极急 · 必立即处理} · 应对:{}
2. 🟡 {候议 · 可缓}

### C 未发现机会(轻提)
- C1 {机会 · 真物锚 · 候用户明示再深耕 · 不自加戏}
```

## 三、3 类项目应用例

### 例 1 · 给开发的代码 review

```markdown
## 工程一刀(对当前 PR #42)
1. 利:测试覆盖率 78% · 比上版 65% 升 · 命名清晰
2. 害:N+1 query 在 line 124 · 1000 用户实测会卡 · 改 batch
3. 守底线:不擅 force push main · 数据库迁移必带 rollback

## 参谋一刀
A 改进:① 加 cache TTL 5min ② 拆 service 到 3 文件
B 风险:🔴 prod migration 未测 dry-run · 🟡 文档 outdate
C 机会(轻提):候 lib 升级时一并 benchmark
```

### 例 2 · 给营销稿审

```markdown
## 凤凰一刀(对小红书稿"深夜 2 点"890 字)
1. 利:创始人 voice 真 · "凌晨 2 点 38 分"具体细节抓人
2. 害:开头 emoji 3 个 · 删 2 个 · 调性更克制
3. 守底线:必标"AI 助理"明示 + 反"必涨粉"承诺(广告法)

## 参谋一刀
A 改进:① 加 1 张真截图 ② 结尾 CTA 由"必看"改"想试可问"
B 风险:🟡 价格 ¥99 vs 历史 ¥YY 不一致 · 必跨档对齐
C 机会(轻提):候续篇可写"凌晨 4 点 AI 怎么回客户"
```

### 例 3 · 给战略提案

```markdown
## 战略一刀(对 v2.0 涅槃方案)
1. 利:数据条件齐(月单 ≥ 10 + LTV 有)· 涅槃触发对
2. 害:14 模块全上 = 反完美主义 · P0 守 5 模块先
3. 守底线:不跳阶涅槃(P0→P2 跳)· 守 30% 改原则

## 参谋一刀
A 改进:P0 先 5 模块 · P1 加 5 模块 · P2 满 14
B 风险:🔴 团队 3 人 14 模块 1 月 = 透支 · 🟡 用户反馈未收集
C 机会(轻提):候 v3.0 加 arXiv 论文升维(P3+ · 不强求)
```

## 四、5 大铁律(双刀必守)

1. ❌ **不阿谀**:grep "完美 / 必然 / 唯一" = 0
2. ❌ **不挑亮点**:害 ≥ 1 / 必带 · 不"全是优点"
3. ❌ **不堆万字**:一刀 ≤ 50 字 / 条 · 参谋 ≤ 100 字 / 段
4. ❌ **不假装做**:每条带真物锚 · 不空泛
5. ❌ **不自加戏**:C 机会默认轻提 · 候明示再深耕

## 五、自检(每发双刀前)

- [ ] 利 ≥ 1 / 害 ≥ 1 / 守底线 ≥ 1 齐?
- [ ] 真物锚 ≥ 1 / 条?
- [ ] 风险标 🔴 极急 / 🟡 候议?
- [ ] C 机会"轻提" · 未自加戏深耕?
- [ ] 反 AI 9 通病 9/9 过(详 02-反AI9通病自检-公开版.md)?

→ 5 项齐 = 双刀过
