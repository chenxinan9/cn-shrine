## 一、N 红线制度

### 1.1 什么是红线?

**红线 = 不能越的合规底线**。任何 1 神碰一下 → 立即停 · 你拍 · 不能商量。

### 1.2 红线怎么定?(你填)

```
红线 1: <合规法规相关>
  含义: <一句话定义>
  禁:   <什么场景下绝对禁止>
  放:   <什么场景下可以提到>
  
红线 2: <价格透明相关>
红线 3: <账号备案相关>
红线 4: <数据安全相关>
红线 5: <身份明示相关>
红线 6: <用户隐私相关>
红线 7: <知识产权相关>
... 你根据业务自己定 N 条
```

### 1.3 实战例(脱敏)

某 SaaS 项目 7 红线:

| 红线 | 含义 |
|---|---|
| 1 | 行业广告法禁词 — 宣称时绝对禁 · 自然语言可 |
| 2 | 价格透明 — 配置包不涉价格 · 价格走独立通道 |
| 3 | 账号备案合规 — 上线前必备 |
| 4 | 数据本地 — 不依赖第三方 |
| 5 | 不冒充真人 — AI 角色必须明示身份 |
| 6 | 隐私法合规 — 不滥用用户数据 |
| 7 | 知识产权 — 自己写 · 不抄 |

### 1.4 红线怎么守?

- **悟空** 干活前看红线清单
- **白泽** 独立查红线有没有被碰
- **你** 红线被碰 = 立即停 · 不商量

## 二、串味三维扫描 🌟

> **原创方法论**。其他 AI 协作框架未见此机制。
> 这是 21 神制衡法的真护城河之一。

### 2.1 什么是串味?

**串味 = 不同类型 / 不同维度的内容意外混进对方场景**。

例:策略文档应该有步骤 · 但写出来像心得感悟 = 类型串味。

### 2.2 三维扫描

#### 维度 1 · 任务 → 反思 串味

- 任务类输出物变成感悟类
- 反例:"怎么做"文档,写满"心得 / 领悟 / 抒情"
- 怎么判:
  - 读完后客户感觉**"我知道下一步干什么"= ✅**
  - 读完后客户感觉**"嗯说得有道理"= ❌**(串味了)

#### 维度 2 · 任务 → 任务 串味

- A 任务的输出物像 B 任务
- 反例:任务 A 应该写"用户增长策略" · 但内容像"用户运营 SOP"
- 怎么判:看输出物的"类型签名"跟任务单是不是对得上

#### 维度 3 · 单元 → 单元 串味

- A 业务的 example 出现 B 业务的场景词
- 反例:餐饮业 example 出现"教培学员"等关键词
- 怎么判:在 A 单元里 grep B 单元关键词 = 0 才算干净

### 2.3 怎么判 4 个类型?

| 类型 | 关键特征 | 反例(串味)|
|---|---|---|
| **任务类**(怎么做)| 有步骤 / 模板 / 操作 | 全是抒情 |
| **反思类**(怎么想)| 有感触 / 领悟 / 思考 | 全是 SOP |
| **数据类** | 有数字 / 计数 / 真物 | 全是观点 |
| **输出类** | 有交付物 / 路径 | 全是流程 |

### 2.4 串味扫描怎么走?

```
悟空交活 → 每个输出物自己标类型
白泽独立查:
  ① 任务 → 反思 串了几条
  ② 任务 → 任务 串了几条
  ③ 单元 → 单元 串了几条
出建议书 → 你拍
```

具体怎么扫:`examples/04-串味扫描方法.md`
