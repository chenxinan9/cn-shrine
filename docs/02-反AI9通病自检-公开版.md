# 反 AI 9 通病自检 · 公开版 SOP

> 你用 Claude / Cursor / Codex / your-llm / 其他 AI 时 · 9 大常见坑 + 预防铁律 · 每大产出前必跑

## 一、9 通病速查表

| # | 通病 | 现象 | 预防 / 自检 |
|---|---|---|---|
| **1** | **堆万字** | AI 给 5000 字 / 你只想要 200 字答案 | 单审 ≤ 200 字 · 多用表格 |
| **2** | **闭门造车** | AI 不看真物 · 凭"我觉得" | WebFetch + 真访 + 路径验证 |
| **3** | **阿谀** | "您说得太对了" / "完美" / "必然" | grep "完美 / 必然 / 唯一 / 一定" = 0 |
| **4** | **假装做** | 写了文件就算完 / 没真物验证 | 真物 ≥ 3 / 自检 9/9 过 |
| **5** | **大词** | "元认知架构师" / "范式重构" 空话 | 不用大词 · 真物驱动 |
| **6** | **玄学** | "感觉 / 直觉 / 气质" 不可量化 | 必带数据 / 真物 / 客观锚 |
| **7** | **越界执行** | 帮你直接发了 / 改了不该改的 | 重大方向必反问 · 不自定 |
| **8** | **完美主义** | 改 100 遍才肯交付 | 守 30% 改原则 · 反钟摆 |
| **9** | **5 元素自检表癖** | 堆"X/10 + 5 元素自检表"形式 | 一刀 1-3 条具体建议 · 不堆 |

→ **9/9 过 才交付** · 任意 1 个 ≤ 7/10 = 退回 V{N+1}

## 二、自检脚本(可跑 · bash)

```bash
#!/bin/bash
# 反 AI 9 通病 grep 自检
# 用法: ./check.sh <你的草稿.md>

FILE="$1"
echo "=== 反 AI 9 通病 grep 自检 ==="

# 通病 3 · 阿谀
echo "--- #3 阿谀 ---"
grep -E "完美|必然|唯一|一定|绝对|肯定" "$FILE" | head -5

# 通病 5 · 大词
echo "--- #5 大词 ---"
grep -E "元认知|范式重构|颠覆|彻底改变" "$FILE" | head -5

# 通病 1 · 堆字
echo "--- #1 字数 ---"
wc -m "$FILE"
echo "(单审 ≤ 200 字 / 整文 ≤ 2000 字 较优)"

# 通病 6 · 玄学
echo "--- #6 玄学 ---"
grep -E "感觉|直觉|气质|大概可能|应该" "$FILE" | head -5

# 通病 9 · 自检表癖
echo "--- #9 自检表癖 ---"
grep -c "自评" "$FILE"
echo "(自评 ≤ 1 处)"

echo ""
echo "=== 全 0 命中 = 通过 ==="
```

## 四、按场景应用

### 场景 1 · 写代码

```
0 · date 真跑
1 · 听完(完整 spec · 不漏)
2 · 理解(主要 + 潜在边界)
3 · 反方测压(若有矛盾 · 列 ≤ 3)
4 · 反问澄清(重大方向必反问)
5 · 才动手
6 · 收口(测试 + 自检 9/9)
```

### 场景 2 · 写文案

- 真物 ≥ 3(真客户原话 / 真数据 / 同行对标)
- 反 AI 味 grep("震惊 / 必涨粉 / 改变一切" = 0)
- 反夸张数字(广告法守)
- 反冒充客户(必标"AI 助理"若 AI 互动)

### 场景 3 · 做战略 / 提案

- "参谋一刀" 3 件套:A 改进 / B 风险 / C 机会(轻提)
- 重大方向必反问user · 不自加戏
- 自评下调允许 · 不阿谀
