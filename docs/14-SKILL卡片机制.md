# SKILL 卡片机制 · 经验怎么沉淀

> 每踩 1 坑 / 发明 1 方法 = 记 1 张卡片 · 20 张攒起来 = 办法手册

---

## 一、干嘛要记?

实战痛点:

- 跑了 100 轮 · 踩了 50 个坑
- 但聊久了忘了 · 下次又踩一样的坑
- "我以后注意" = 空话 · 不记盘 = 必重犯

解药:**每个坑记 1 张卡片** · 写盘 · 跨会话保留 · 永久。

---

## 二、卡片模板

| 字段 | 填什么 |
|---|---|
| **序号** | SKILL-XXX |
| **名称** | 4-8 字简称 |
| **立日** | YYYY-MM-DD |
| **立人** | 哪个神立的 |
| **什么时候用** | 列 1-3 个场景 |
| **怎么用** | 列 1-3 条铁律 |
| **从哪来** | 时锚 + 实战例 + 你的原话(如果是你纠的)|
| **谁能用** | 哪些神 / 哪些项目 |
| **配什么** | 跟哪些 SKILL 配 |
| **防什么** | 防哪个 AI 通病 |
| **重犯前查什么** | 发回复前 grep 什么 |

## 三、实战例

### 卡片 1 · 自报必复查

| 字段 | 填什么 |
|---|---|
| 序号 | SKILL-001 |
| 名称 | 自报必复查 |
| 立日 | 2026-0X-XX |
| 立人 | 白泽 |
| 什么时候用 | 任 1 神交自查报告时 / 跨身份交接前 / 上线前最后一查 |
| 怎么用 | 1. 不接受免查 · 自查 ≠ 终审<br>2. 白泽独立查关键字<br>3. 自报数和真物数对账 · 差值 = 真盲区 |
| 从哪来 | 实战:悟空自报"清完 N 条" · 白泽查出 N+10% · 你原话:"不信自报 · 只信真物" |
| 谁能用 | 全身位 · 任 1 神自报必复查 |
| 配什么 | 反 AI 9 通病自检 + 双审闭环 |
| 防什么 | 第 4 通病(假装做)+ 第 7 通病(漏报)|
| 重犯前查 | "悟空已自查 OK 直接 push" / "跳过白泽审" = 0 才发 |

## 四、什么时候必记?

✅ 记:
- 踩到 1 个坑(包括小坑)
- 发明 1 个新方法
- 你明示纠了 1 次
- 同类问题第 2 次发生(警告)
- 同类问题第 3 次发生(必记)
- 跨身份大冲突解决后

❌ 不记:
- 一次性琐事
- 已有 SKILL 覆盖
- 没可复用性

```
    SKILL-002-不擅 rm.md
    SKILL-003-串味三维扫描.md

```

```
SKILL-[3 位序号]-[4-8 字简称].md
```

### 升级路径

```
v1.0 草拟 → 实战验 → v1.1 完善 → 跨神普及 → v2.0 抽象通用
```

## 七、跟其他记忆系统的关系

- **superpowers**(skill 系统) — 通用 skill 集 · 跨项目复用
- **claude-mem**(memory 系统) — 时序记忆 · 自动捕捉跨会话
- **本卡片机制** — 项目特化 · 手动沉淀 · 高质量

三者关系:**互补不互斥**。

- 通用方法用 superpowers
- 时序记忆用 claude-mem
- 项目方法论用本机制
