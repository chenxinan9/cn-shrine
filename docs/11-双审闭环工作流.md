# 双审闭环工作流

> 21 神制衡法的工作流落地。任何要量产的活,必走这条链。

---

## 一、8 步流水线(含太白起点)

```
想策略 → 派任务 → 干活 → 自查 → 独立查 → 你拍 → 修 → 复查 → 定稿
 ↑       ↑      ↑     ↑      ↑       ↑     ↑    ↑     ↑
太白    唐僧   悟空  悟空   白泽    你   悟空  白泽   你🔒
```

8 步 · 5 神 · 双审 = 自查 + 独立查

---

## 二、为什么"自查"不能当终审?

实战例(脱敏):

- 悟空自报"已经清完 N 条违规词"
- 白泽自己跑一遍 → 找出 N+10% 真残留
- 差的 10% = 自查的盲区

**结论**:
- 自查有价值(快速回收 90%)
- 但不是真值 · 必须另一个神再独立查一遍

## 三、8 步详解

### 第 0 步 · 想策略(太白)

你说要干个事,太白先给:
- ABC 三个方向 + 每个的利害
- 给悟空 / 白龙的提示词模板
- 候你拍 A / B / C

### 第 1 步 · 派任务(唐僧)

你拍完方向,唐僧把活拆细,派给悟空。

派单内容:
- 干啥(任务)
- 用什么提示词(太白给的)
- 用什么规范(沙僧的标准)
- 死线
- 必守的禁区

### 第 2 步 · 干活(悟空)

悟空根据任务单开干。

交付:
- 主体真物(代码 / 数据 / 文档)
- 真物路径

### 第 3 步 · 自查(悟空)

悟空交付前先自查:
- 反 AI 9 通病(详 `02-反AI9通病自检-公开版.md`)
- 任务单每条要求逐项打勾
- 自报"已 PASS" + 真物路径

⚠️ 自查报告不算定稿 · 必须进下一步。

### 第 4 步 · 独立查(白泽)

白泽**不准看代码细节** · 只干 4 件:

- 查真物
- 数数(grep 关键字)
- 跟自查数对账
- 找差值 / 找茬 / 说利害

交付:**建议书**(.md)· 里面写清:

- 真物数:X
- 悟空自报数:Y
- 差值:X - Y = ?
- 找茬:列具体几条
- 利:值得保留的
- 害:必须改的
- 建议怎么改(但不动手)

### 第 5 步 · 你拍(你)

你看建议书 → 拍:

- A · 接受白泽找茬 · 让悟空改
- B · 部分接受 · 局部改
- C · 反驳白泽 · 维持原样

不擅替你拍。你是最后的人。

### 第 6 步 · 修(悟空)

悟空根据你的拍板修。

⚠️ 白泽**不能修** · 守第 5 条铁律(找茬的不修)。

### 第 7 步 · 复查(白泽)

白泽再查一遍,验修完的真物。

交付:复查报告(差值应该 = 0 或接近 0)。

### 第 8 步 · 定稿(你)

你看复查报告 → 打个稳定标签:

```
定稿-vX.X-日期
```

锁定后任何改动都要开新版本号 · 不能覆盖。

## 四、5 神在 8 步中的"干"与"不干"

| 神 | 干 | 不干 |
|---|---|---|
| **太白** | 第 0 步想策略 | 不动手 / 不审 / 不拍 |
| **唐僧** | 第 1 步派任务 | 不写策略(用太白)/ 不替悟空 |
| **悟空** | 第 2/3/6 步干活 + 自查 + 修 | 第 4 步不审自己 / 第 5 步不替你拍 |
| **白泽** | 第 4/7 步独立查 + 复查 | 不写代码 / 不动手修 |
| **你** | 第 5/8 步拍板 + 定稿 | 不替 AI 干 / 不替 AI 审 |

## 六、什么时候可以跳步?

| 活多大 | 最小链路 |
|---|---|
| 1 小时内单人小事 | 第 2 步 → 第 3 步 → 第 8 步(自查即定稿)|
| 1-4 小时中等 | 第 2 → 3 → 4 → 5 → 8 步(单审)|
| 4 小时以上要量产 | 8 步全跑(双审)|
| 红线 / 客户面 / 合规 | 8 步全跑 + 多神交叉审 |

判别:**活越大 / 风险越高 / 越要全 8 步**

## 七、反"双审被简化" 3 通病

| ❌ 通病 | ✅ 正解 |
|---|---|
| "悟空自查过了 · 直接 push" | 小活可以 · 量产必白泽再查 |
| "白泽找茬太多 · 我不管" | 不接受 = 反第 4 条铁律 · 你必逐条拍 |
| "白泽顺手帮我修了" | 反第 5 条铁律(找茬的不修)· 违反审计独立 |
