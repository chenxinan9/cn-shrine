# 成本与效率记录模板

> 每个阶段必记。将来分享 / 复盘 / 议价时,这组数字最有说服力。

---

## 一、干嘛要记?

实战痛点:

- 跑完不知道"花了多少时间 / 多少钱"
- 议价不知道"我贡献了什么"
- 复盘不知道"哪步效率低"
- 分享不知道"故事怎么讲"

解药:**每个阶段记 7 项**(下面)· 跑完一目了然。

---

## 二、7 项记什么

| 项 | 记什么 |
|---|---|
| **阶段名 + 时间** | 例:V0.5 量产 · 跑了 48 小时 |
| **AI 神数** | 例:7 神(太白 + 悟空 + 白泽 + 唐僧 + 千眼 + 白龙 + 沙僧)|
| **轮次** | 干活 N 轮 + 审 N 轮 + 修 N 轮 |
| **你拍板次数** | 大决策 / 红线 / 冲突 一共拍几次 |
| **总耗时** | 墙钟时间 X 小时 + AI 跑动时间 Y 小时 |
| **你实际投入** | 转 prompt Zh + 思考 Wh = 真你时间 |
| **最后出了啥** | 量 + 质(9 通病过几个 / 红线 0 / 串味 0)|

---

## 三、实战例(脱敏)

| 项 | 内容 |
|---|---|
| 阶段 | V0.5 内容生产线 · 48 小时 |
| AI 神数 | 7 神(太白 + 悟空 + 白泽 + 唐僧 + 千眼 + 白龙 + 沙僧)|
| 轮次 | 悟空干 3 轮 + 白泽审 4 轮 + 修 2 轮 |
| 你拍板次数 | 3 次(三大冲突)+ 红线触发 0 |
| 总耗时 | 墙钟 48h + AI 跑动 44h |
| 你实际投入 | 转 prompt 2h + 看建议书 3h = 5h |
| 最后出 | P 件真物 · 9/9 通病过 · 7/7 红线过 · 串味 0 |

## 四、3 个故事化角度

### 4.1 给投资人

```
1 个独立创业者 + 7 个 AI 角色,48 小时,
出 P 件量产级东西,反 AI 9 通病 9/9 过,
N 红线 0 触发,串味 0 中。

跟传统外包对比:
  外包:同量需 N 人 × T 周 + Y 万元
  我们:1 人 + 7 AI + 48h + 5h 真你时间
```

### 4.2 给同行 SMB 老板

```
不需要懂代码,也能用 AI 做内容生产线。
我用 21 神制衡法跑了 N 项目,总结 4 件:
  1. AI 互相审,真物级(跟外包不一样)
  2. 你只需要拍板和转 prompt
  3. 一周可学会
  4. 试 1 个小活,见证后再推大用
```

### 4.3 给 Z 世代

```
你不用学复杂框架,不用调 API,不用 prompt engineer。
你只需要用 21 神制衡法,选 3-7 个神,跑双审闭环。
我用这个办法,1 个人公司也能跑出量产级东西。
```

## 五、记录工具(轻量)

不需要复杂工具,1 个 markdown 文件足够:

```
your-project/
  成本/
    YYYY-MM-阶段X-成本效率.md
```

或用 Notion / Excel / Google Sheet 跟踪。
