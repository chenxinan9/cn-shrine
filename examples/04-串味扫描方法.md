## 一、串味三维回顾

| 维度 | 内容 | 怎么判 |
|---|---|---|
| **维度 1 · 任务 → 反思** | 任务类输出物变成感悟类 | 读完客户感觉"我知道下一步干什么"= ✅ / "嗯说得有道理"= ❌ |
| **维度 2 · 任务 → 任务** | A 任务输出像 B 任务 | 输出的类型签名 vs 任务单是否对得上 |
| **维度 3 · 单元 → 单元** | A 业务 example 出现 B 业务场景词 | A 单元里 grep B 关键词 = 0 才干净 |

---

## 二、白泽手动扫描(不会写脚本也行)

### 维度 1 · 手动扫

```bash
# 任务类文档里查反思关键词
grep -E "心得|感悟|领悟|我觉得|其实|想想看|深思|反思" your-project/tasks/*.md | head -20

# 命中 > 0 = 可能串味 · 人工复核
```

### 维度 2 · 手动扫

```bash
# 任务 A 文档里查任务 B 的关键词
grep "B 任务关键词" your-project/tasks/task-A-*.md

# 命中数高 = 类型签名错位
```

### 维度 3 · 手动扫

```bash
# 业务单元 A 的 examples 里查 B 的场景词
grep "B 业务场景词" your-project/examples/unit-A/*.md

# 命中 > 0 = 串味
```

## 三、4 个类型怎么判?

| 类型 | 关键特征 | 反例(串味) |
|---|---|---|
| **任务类**(怎么做) | 有步骤 / 模板 / 操作 | 全是抒情 |
| **反思类**(怎么想) | 有感触 / 领悟 / 思考 | 全是 SOP |
| **数据类** | 有数字 / 计数 / 真物 | 全是观点 |
| **输出类** | 有交付物 / 路径 | 全是流程 |

## 四、扫描报告模板

```markdown
# 串味三维扫描报告

立日: YYYY-MM-DD HH:MM
扫人: 白泽
项目: <项目名 · 版本号>

## 维度 1 · 任务 → 反思 串味
- 扫描文件数: N
- 命中数: M
- 详情:
  - <文件路径>: <类型> · <hits> 命中

## 维度 2 · 任务 → 任务 串味
- 扫描文件数: N
- 命中数: M
- 详情:
  - <文件路径>: <从 X 写成 Y>

## 维度 3 · 单元 → 单元 串味
- 扫描文件数: N
- 命中数: M
- 详情:
  - <文件路径>: <A 出现 B 场景词> · 泄露词: [...]

## 总结
- 总串味数: <数>
- 判决: PASS / FAIL
- 候你拍板修复优先级
```

## 五、扫描流程

```
悟空交活 → 每个输出物自己标类型
   ↓
白泽独立扫:
  ① 任务 → 反思 串了几条
  ② 任务 → 任务 串了几条
  ③ 单元 → 单元 串了几条
   ↓
出建议书 → 你拍板
   ↓
悟空修(若有串味)
   ↓
白泽复查
   ↓
你定稿
```

## 七、起手 3 步

### 第 1 步 · 整理关键词库

按 4 个类型 + 业务单元,整理一份关键词库:

```
关键词库:
  任务类: [步骤, 操作, 执行, ...]
  反思类: [心得, 感悟, 领悟, ...]
  单元 A: [A 场景词 1, A 场景词 2, ...]
  单元 B: [B 场景词 1, B 场景词 2, ...]
  ...
```

### 第 2 步 · 跑 1 次手动扫

用 grep 走维度 1/2/3 一遍 · 看命中数。

### 第 3 步 · 出建议书

按报告模板出 1 份建议书 · 候你拍。
