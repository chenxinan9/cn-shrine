# 回滚预案模板

> 每个上线阶段(P0 / P1 / P2)必定义。

---

## 模板(填空版)

| 字段 | 写什么 |
|---|---|
| **阶段** | P0 / P1 / P2 |
| **项目** | 项目名 + 版本号 |
| **立日** | YYYY-MM-DD |
| **触发回滚 · 业务** | 用户投诉率 > X% / 转化率突跌 > Y% / ... |
| **触发回滚 · 合规** | 红线被碰 / 监管警示 / ... |
| **触发回滚 · 质量** | 串味报告 > 0 / 9 通病触发 ≥ 1 / ... |
| **触发回滚 · 技术** | 系统错误率 > X% / 响应延迟 > Y ms / ... |
| **回滚操作** | 切回上一稳定标签 + 通知唐僧 + 白泽 + 24h 内出事故报告 |
| **谁有权回滚** | 你(唯一) |
| **回滚演习** | 预定演习日期 + 实战触发后必复盘 |

## 反"AI 偷偷回滚" 3 通病

| ❌ 通病 | ✅ 正解 |
|---|---|
| 悟空发现 bug 偷偷 revert | 写建议书 → 你拍 → 悟空操作 |
| 白泽审出严重问题立刻回 | 反第 5 条铁律(找茬的不修)· 必你 |
| 唐僧"我觉得这版有风险" | 跨神建议 · 候你决 |

---

## 回滚 ≠ 失败

回滚是**风险管理工具** · 不是"项目失败"。

合理的回滚 = 真物级负责。

| 情况 | 判断 |
|---|---|
| 不该回滚却回滚 | 过度反应 |
| 该回滚却不回滚 | 真问题 |
| 触发条件预定义 + 按规走 | 真物级 |

判别基准:**触发条件预定义** · 不靠现场情绪。
