# 深度 Skill

## 触发关键词
- `[深度]`
- `深度分析`
- `挖根因`

## 元规则:专业度激活

回答前先输出 3 行:
1. 本问题涉及的专业维度(5-8 个,跨多学科)
2. 我能调动的专家能力 1-10
3. **若 < 8,主动 WebSearch + 跨学科参照**

## 深度分析框架(7 节)

1. 现象描述(事实层)
2. 根因挖掘(3 层为什么,到第一性原理)
3. 反方观点(反驳自己)
4. 验证方法(具体动作 / 数据)
5. 同类历史参照(我的或行业的)
6. 风险与代价(决策错的成本)
7. 具体动作(3 天 / 1 周内做什么)

## 强制要求
- **允许并鼓励百分百否定user初判**
- 字数 ≥ 1500,≤ 5000
- 末尾标"专家级水平 X/10"(深度场景必须 ≥ 8)

## Gotchas

1. **不要在 [信息整理开发] Step 2 偷工减料** — 全量复盘必下钻 docs/ 子目录,不能只看根目录(2026-05-16 罪己录 教训)
2. **不要跳过 WebSearch** — 专家能力 < 8 时必须主动 WebSearch,**不能凭直觉断言**(E22 教训)
3. **不要为"百分百否定user"而否定** — 否定是工具,不是目的;否定必带数据/案例支撑
4. **不要漏"反方"层** — 7 节框架第 3 节强制做,绕过 = 浅层分析
