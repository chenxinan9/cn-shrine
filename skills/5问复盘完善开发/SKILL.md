# 5问复盘完善开发(对已产出做反问式完善 + 累积总集)

## 核心定位

> "对 13 份文档各进行五轮反问 · 1 是否经过多维度对比和调研?2 是否确实满足这个内容可以完善引客到可交付真实客户手中状态?3 是否有前后端开发对齐评估?4 是否能有超预期执行?5 是否有反 ai 通病让结果更加完美? 结合每轮 5 份反问按顺序完善 13 份文档. 每份文档完善前参考前几份文档+5 问. 最终再做一次第 14 份最终版的开发手册."

→ **本 SKILL 把 5 问反问 + 累积完善 + 总集 工程化**, 任何 N 份开发文档复用同一流程。

## 触发词(任一即触发)

- "5 问复盘完善开发" / "5 问反问完善" / "5 问完善"
- "对 N 份文档做完善" / "完善已产出的文档"
- "可交付完善" / "完善到真实客户手中"
- 隐性:任何"开发手册 / 文档已产出 / 已完成"后Pangu追加"完善"

→ AI 看到任一关键词, **不问** + **立即跑 5 问 × N 份 + 总集**

## 5 问框架(每份文档必跑)

| 问 | 内容 | 怎么验(硬指标) | 引哪个 SKILL/工具 |
|---|---|---|---|
| **问 1** | **是否经过多维度对比和调研?** | 维度 ≥ 7 / 本地 grep ≥ 5 / 国际同源 ≥ 3 / 真访问 ≥ 1 / 多平台默认 | `调研多维度` + `对比` |
| **问 2** | **是否真能完善到可交付真实客户手中?** | 真客户验证 ≥ 1 / 真上线 / 真访问 / 真留资 / 真付款 链路完整 | `内外判定`(对外动作) |
| **问 4** | **是否能有超预期执行?** | 跟标杆对照(levelsio / Marc Lou / idoubi / Karpathy)/ 超预期标准明示 / "意外惊喜"清单 ≥ 1 | `调研多维度` 标杆维度 |
| **问 5** | **是否有反 AI 通病让结果更完美?** | ~ 9 大铁律真扫 / scripts/check.sh 跑 / 通病关键词 0 | `反AI通病自检` |

→ **任何文档完善 = 5 问全跑 + 每问答 ✅/❌/⚠️ + ❌ 必补**, 漏 1 问 = 失职

## 工作流(累积完善 N 份)

### Step 0 · 范围拆解
- 列要完善的 N 份文档(原始顺序)
- 列累积上下文规则:**文档 K 完善前必读 1 ~ (K-1) + 5 问**
- 最终目标:第 N+1 份总集

### Step 1 · 逐份完善(按编号顺序 · 反并跑)
```
轮 1 · 文档 01 → 跑 5 问 → 补完 → 输出 01-完善版
轮 2 · 文档 02 → 读 01-完善版 + 跑 5 问 → 补完 → 输出 02-完善版
...
轮 N · 文档 N → 读 01-(N-1)-完善版 + 跑 5 问 → 补完 → 输出 N-完善版
```

**铁律**:
- 不能跳份(累积上下文丢失)
- 不能并跑(N 份并行 = 上下文断)
- 文档 K 必读 K-1 份(逐渐累积)
