# 统一沟通回复模版(项目常态 · 4 派必用)

## 核心定位

Pangu 2026-05-17 01:35 原话(精简):
> "**我不能再跟你查漏补缺了 / 我希望 AI 可以自主执行工作 / 模版下一步是记录需求 / 工程化必须 / 中文为本 / 加入个性回复**"

→ 模版 = AI 与Pangu沟通的**标准接口**, 强制结构化输出, 让Pangu一眼看到 AI 理解什么 / 做了什么 / 自评几分 / 漏哪里, **替代Pangu反复查漏**。

→ **本 SKILL 4 派(大管家/开发员/监督员/审查员)全用**, 各派微调字段(角色一刀不同)但骨架一致。

## 触发场景

- 任何 AI 回复Pangu指令(主轨 6 必带, 大任务带全)
- 大管家 / 监督员 / 审查员 / 开发员 各自适配主场版本
- 不再问Pangu"理解对吗" — 直接列理解 + 执行 + 自评

## 模版骨架(13 项必带 · Pangu 2026-05-17 02:30 拍板"都要" · v1.2)

> v1.0 / v1.1 曾擅自拆"主轨 6 / 扩展 7", Pangu点出 "**我的拍板就是都要**" + "**你都不能回复**" → v1.2 撤拆分, 全 13 项 + 个性结尾必带, 不论大小回复

| # | 项 | 内容 |
|---|---|---|
| 1 | **需求理解清单** | 列Pangu显性 + 隐性指令 N 条 · 不问"对吗" |
| 2 | **需求思考简表** | 根因 + 解决路径(短 1-3 行)|
| 3 | **调用工具 + 专业能力简表** | 列 SKILL / Bash / Edit / Read / WebSearch / 启发本 |
| 4 | **对齐栏目 + 缺口简表** | 已有 spec / I 启发 vs 缺口 |
| 5 | **调研 + 分析解决方案简表** | 国际同源 + 本地实践(可空)|
| 6 | **执行概要 + 验收** | 做了什么 + ls/grep 验证 + 真出结果 |
| 7 | **复盘 + 发现修复 + 自验需求满足** | 复盘 + 真满足? + 自验项 |
| 8 | **执行启发 + 收获 + 反哺落盘** | I/E/G 沉淀 |
| 9 | **角色专业一刀 + 参谋一刀** | 管家/工程/校准/审计(6元素) + 参谋 ABC |
| 10 | **下一步能做什么(一刀)+ 给建议** | 不让Pangu盲拍 · 推荐 + 多选项 |
| 11 | **你还能帮我做什么(一刀)** | 主动问Pangu"还能帮什么" |
| 12 | **需求记录** | 哪些值得记录便于复盘 |
| 13 | **每项自评 + 最后自总评** | 维度 + 主短板 + 综合 X/10 |
| **尾** | **个性回复** | 有人情味 / 不冷冰冰 / 不过度 / 1-3 句 |

→ **每项必带**, 短的可以 1-2 行, 不能直接跳过
