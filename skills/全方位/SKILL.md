# 全方位 Skill

## 触发关键词
- `[全方位]`
- `360 度评估`
- `不要漏角度`

## 元规则
开始前输出 10 个维度的专业能力自评(1-10),<6 的维度必须 WebSearch。

## 10 维度逐项扫(每项 ≥ 80 字)
1. 技术 - 实现难度 / 工具栈 / 沉淀
2. 商业 - 模式 / 定价 / 客单 / 复购 / 利润
3. 客户 - 谁 / 在哪 / 付费意愿 / 决策周期
4. 竞品 - 1-3 个对手 / 差异化
5. 法务合规 - 备案 / 数据 / 内容 / 营业范围
6. 运营 - 一人扛几客 / 自动化省多少
7. 营销示例 - 渠道 / 转化
8. 时机 - 现在 vs N 月后 vs N 年后
9. 风险 - 最坏情况 / 能承受吗
10. 我自己 - 开心吗 / 擅长吗 / "心理上想做"陷阱?

每维度末尾标"该维度专家水平 X/10"

## 综合输出
- ✅ 该做的理由 ≥3 条
- ❌ 不该做的理由 ≥3 条
- 🎯 最终建议(做 / 不做 / 推迟 / 缩小)
- 📋 下一步具体动作(7 天内做什么)
- 整体"专家级水平 X/10"

## Gotchas

1. **不要忽略反例** — 10 维必有"反方",每维至少 1 个反例支撑(E22 教训)
2. **不要假设 18/10 维都有数据** — 没数据的维度必标"⚠️ 待补数据",不空话
3. **不要让"我自己"维度被遮蔽** — 第 10 维"心理上想做"陷阱 = 容易自欺,触发"全方位"必直面
4. **不要把"全方位"做成万言书** — 每维 ≥ 80 字,但 ≤ 200 字;总长度 ≤ 3000 字(反膨胀)
5. **不要在 ≥ 6 个维度 < 6 时还不 WebSearch** — 元规则明定,违反 = SKILL 失效
