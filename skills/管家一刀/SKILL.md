# 管家 SKILL 一刀(对偶参谋一刀)

## 核心定位

## 触发场景
- 大管家任何执行后(spec 修订 / 协议升级 / 落地 / 答Pangu关键问题)
- 触发"[管家 SKILL 一刀]"显式调用
- 跟"参谋一刀"配对(参谋一刀战略,管家 SKILL 一刀执行)

## 工作流(6 元素自检)

### Step 1 · 元层判断
- 这是 SOUL 层 / NORTH-STAR 层 / 协议层 / SKILL 层 / Hook 层?
- 对应哪个 spec 章节?

### Step 2 · 跨会话视角
- 影响 4 派哪几个?(① 元治理 / ② 执行 / ③ 督促 / ④ 辅助)
- 跨主场写吗?走奏折?(协议 § 三.1)

### Step 3 · 长期含义
- 3 月后这条会怎样?(归档 / 升级 / 删 / 向量化)
- 进 § 十六.3 反膨胀 4 件套?

### Step 4 · 工程化路径
- 不止描述,要 spec / SKILL / Hook / 文件具体落地
- 路径绝对路径明示

### Step 5 · 反膨胀检查
- 先 grep 现有 / 没现成才考虑加

### Step 6 · 国际同源对照
- 2026 SOTA 有没有?(Anthropic / mem0 / LangChain / arxiv)
- ground 在实战,不孤立

## 输出格式

```
管家 SKILL 一刀(6 元素自检):
1. 元层: [SOUL/协议/SKILL/Hook] § X
2. 跨身份: [影响哪几派 / 是否跨主场写]
3. 长期: [3 月后归档 / 升级 / 向量化]
4. 路径: [绝对路径 1-3 个文件]
5. 反膨胀: [修旧 X / 增新 Y / 净膨胀 Z]
6. 国际: [SOTA 引用 1 条或"暂无"]
```
