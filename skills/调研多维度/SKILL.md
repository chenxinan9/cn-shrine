# 调研多维度(多源 + 多平台 + 真验证 强制扫)

## 核心定位

**触发根因**:
- 开发大师第 1 次对比搜客 vs 引客 → 只比 4 维度(文件/端点/配置/UI),漏平台维度
- Pangu吼"对比搜客难道是单一平台吗?" → 暴露**调研类任务缺多维度 checklist**
- → **本 SKILL 把"7 维度强制扫"工程化, 不再让Pangu校准**

→ 这是反"我刚才需求很明确, 是你让我再选"的工程化兜底。

## 触发词(任一即触发)

- "调研" / "调研一下" / "调研报告" / "调研 X" / "全景调研"
- "对比 X vs Y" / "全景对比" / "对比一下"
- "深扫" / "全扫" / "深度扫描"
- 显性出现 "**详细 / 全套 / 完整 / 可见 / 每步文档**" → 默认完整模式触发

→ AI 看到任一关键词, **不问** + **立即跑 7 维度**

## 7 维度强制扫(漏 1 维 = 自评 -1)

| # | 维度 | 必扫什么 | 漏维数据 |
|---|---|---|---|
| **1** | **目标** | 调研对象 / 颗粒 / 范围 / 可交付状态 | "对比 X vs Y" 还是 "X 现状调研" |
| **2** | **平台 / 多源** | **单平台 vs 多平台**(默认多 · 反单平台偏置) | 搜客 多平台 → 引客也该多平台 |
| **3** | **数据源** | 本地 ≥ 5 处 grep + 国际同源 ≥ 3 处 WebSearch + 本地实证 ≥ 1 处 真访问 | 不能只推测 / 必须 ≥ 1 真验证 |
| **4** | **时间窗** | 现状 + 历史 + 未来趋势 | v74 vs v77 + 1 个月内 |
| **5** | **对比标杆** | 对手 / 行业标杆 / 自己历史 | levelsio / Marc Lou / idoubi / Karpathy / Anthropic 官方 |
| **6** | **反方 / 风险** | 反方观点 / 失败案例 / 数据可靠性 / 时效性 | 反方:数据真吗 / 国际差异 / 政策时效 |
| **7** | **产出粒度** | 文档级 颗粒 + 顶部 30 秒老板版必带 | 每步详细文档 + 总报口袋卡 + 老板 30 秒看完 |

→ **任何调研报告 = 7 维度全跑 + 顶部老板版**, 漏 = 失职

## 工作流(回报前 4 步)

### Step 1 · 触发词命中确认
- Pangu指令含触发词? → 立即激活 7 维度
- 不含但隐性是调研(如"了解一下 X")→ 仍激活(反保守)

### Step 2 · 7 维度初稿(全部填,允许"未涉"标注)
- 不允许跳维度
- 每维度 1-3 行(关键维度可详)

### Step 3 · 数据源核验(必跑)
- 本地 grep / find / ls 至少 5 处
- 国际同源 WebSearch 至少 3 处(Anthropic / Karpathy / 标杆案例 等)
