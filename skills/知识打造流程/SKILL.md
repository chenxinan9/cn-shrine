# 知识打造流程 Skill(原 一舟"知识宫殿")

## 触发关键词
- `[知识打造流程]`
- `打造知识库`
- `筛选洞察`
- `定期整理中枢`

## 核心定位

知识中枢 ≠ 文件堆。**是金字塔,不是堆**。
- M-flow(原始流水)= 100% 不筛
- 共享记忆(稳定事实)= 70% 进
- **知识宫殿(战略认知)= ≤ 5% 进,打分 ≥ 95 才入**

→ 大部分"知识"该死在 M-flow 层,不该升级。

## 3 层记忆边界

| 层 | 内容 | 不该放的 | 该放的 |
|---|---|---|---|
| **M-flow** | 原始会话 / 临时想法 / 一次性操作 | (这层无门槛) | session 流水 |
| **共享记忆** | 稳定事实 / 多 agent 共用 | 未验证想法 | 跨项目通用配置 |
| **知识宫殿** | 战略认知 / 长期复用 | 普通问答记录 | 已验证洞察 / 战略决策 / 反复用经验 |

## 打分系统(对应一舟规范)

| 分数 | 星级 | 含义 |
|---|---|---|
| 100 | ⭐⭐⭐ | 核心战略 / 立即重要 / 经验证 / 可直接执行 |
| 95-98 | ⭐⭐⭐ | 重要战略 / 已验证 / 高执行价值 |
| 90-92 | ⭐⭐ | 重要指导 / 有参考价值 / 需调整 |
| 85-88 | ⭐ | 参考级 / 局部适用 / 需结合场景 |
| < 85 | 无 | 参考信息,**不纳入核心认知** |

## 5 步执行流程

### Step 1 · 扫描待分析文件
- 列入待评目录(默认 `~/Desktop/📚 安的知识中枢/_待筛选/`)
- 显示文件列表 + 修改时间
- 没有 → 告知 + 结束

### Step 2 · 深度分析每个文件
- 合并同类项(识别与现有重叠)
- 新增洞察(发现全新战略)
- 打分评估(用上表)
- 冲突检测(与现有认知矛盾的标记)
