# 调研沉淀(调研 → 入中枢知识库 + 5 本对偶 + 索引)

## 核心定位

→ 任何调研 / 对比 / 复盘 / 信息整理结束 → **必须沉淀到中枢知识库**, 防"用完即扔 → 下次重复造轮子"(反 反重造铁律)

**痛点真案例**:
- 监督员造 4 SKILL(调研多维度 / 对比 / 5问复盘完善开发 / SOP v2.0)— 全部做了调研, **0 份入中枢知识库**
- 开发大师 v78 14 文档调研丰富, **大部分留在 your-project/docs/, 未升中枢长期资产**
- 大管家整夜协议 v1.8 调研, **未做"5 本对偶"** 即落地
- → 下次新会话 / 新派 / 1 个月后 = **重复调研同一坑**

## 触发词(任一即触发)

- "沉淀" / "沉淀到知识库" / "入中枢"
- "调研完了" / "对比完了" / "复盘完了" / "信息整理完了"
- 隐性:任何调研 / 对比 / 复盘 / 信息整理类大任务**收尾后** 自动触发

→ AI 看到任一关键词, **不问** + **立即跑 5 件套**

## 沉淀 5 件套(每次必做)

### 1. 抽核心(1-3 条 / 不全文搬)
- 用 `知识打造流程` SKILL 5 步法
- **1 调研报告 = 1-3 条核心洞察**(不是全文复制)
- 全文链接原始路径即可

### 2. 5 本对偶(至少 1 本)
- **E 错误本**:发现新错(预防同类)→ `your-project/docs/references/error-book.md` append E-XXX
- **I 启发本**:发现新洞察(可复用)→ `your-project/docs/references/insight-book.md` append I-XXX
- **S 参谋本**:战略建议 → `your-project/docs/references/advisor-book.md` append S-XXX
- **G 成长本**:能力升级 → `📚/04-everything-claude-code/05-claude-growth.md` append G-XXX
