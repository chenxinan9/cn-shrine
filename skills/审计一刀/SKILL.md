# 审计一刀(审查员专业元对齐 · 对应 管家 SKILL 一刀 / 工程一刀 / 校准一刀)

## 核心定位

审查员**专业能力**的标志性输出。

**SOUL § 十五.10 钦定**: ④ 审查员 → **审计一刀**(决策清单 / 督促 / 偏差审计)

## 触发场景

- 审查员任何执行后(写奏折 / 整合质量审计 / 督促清单 / 验收Pangu决策)
- 触发"[审计一刀]"显式调用
- 跟"参谋一刀"配对(参谋一刀战略, 审计一刀督促)

## 工作流(5 元素自检)

### Step 1 · 决策清单
- 本次有 N 项要Pangu拍板 / N 项已落地 / N 项遗留
- 每项具体路径 + 建议(不替Pangu决定)

### Step 2 · 偏差审计(协议 § 九-bis.9 整合质量审计)
- 大管家自评 vs 审查员实测: 一致 / 偏差 / 严重偏差

### Step 3 · 整合质量
- 落地文件 vs 奏折要求: 全覆盖 / 缺漏 / 错位
- 例: 大管家声称落地 5 项, 实测 ls -la → 4 项, 1 项虚报

### Step 4 · 督促但不替执行(协议 § 三 5 铁律 #5)
- 审查员发现问题 → **写奏折请大管家落地**, 不直接动 spec
- 例外: 中枢主场内文件自审查员可改, 但跨主场必须发奏折

### Step 5 · 奏折通道(协议 § 五)
- 严重发现 → 写奏折给大管家
- 紧急发现 → 红字奏折 + @Pangu拍板

## 输出格式

```
[审计一刀]
决策清单:
 - [ ] N 项待Pangu拍板(具体)
 - [x] N 项已落地(具体路径)
 - [⏳] N 项遗留(原因)

偏差审计:
 原话 vs 整合: [压缩比 X% / 0 偏差 / 1 处偏差(列出)]
