# 复盘补缺自我迭代(7 阶段闭环 SOP)

## 核心定位

> "**说不定你还有新发现呢. 结合这些复盘, 你回过头来查漏完善, 也许有很多不需要完善的. 你先复盘包含自己的回复复盘, 罗列我的需求和你承诺, 对照我的回复是否做到, 查漏后分析, 因为可能有一些历史问题已经解决或者没有那么重要, 先整体做一遍, 然后再发现对齐是否真的有缺口, 和做各项补缺必要性和补缺计划, 然后再深度分析和调研如何补缺, 再执行, 再自我升级. 请你做好前置规划和细节执行, 最终给我一个结果清单**"

→ 不是简单"自查 + 修", 是**结构化 7 阶段闭环**:
- Phase 1 整夜对话复盘(信息整理开发 Step 1-3 骨架)
- Phase 2 查漏分析(分真缺 / 已解 / 不重要)
- Phase 3 补缺必要性 + 计划(P0/P1/P2 排序)
- Phase 4 深度调研(国际同源 + 本地实践)
- Phase 5 执行(并行落地)
- Phase 6 自我升级 + 结果清单
- Phase 7 项目反哺(user"别忘项目")

## 触发场景

- 用户明示"[复盘补缺自我迭代]"
- user说近义触发词: "全量复盘 / 查漏补缺 / 自我迭代 / 整体做一遍 / 复盘 + 反哺"
- 大管家自检发现"3 次同类犯 + 漏东西不闭环"( 三次为定律)
- 大任务回复结尾(管家 SKILL 一刀 6 元素 + 参谋一刀 3 件)前先跑一遍

## 工作流(8 阶段串行 · 不跳 · Step 0 主动问范围)

### Step 0 · 主动问user取值范围(v0.2 新 · 必走)

→ 触发 [复盘补缺自我迭代] 后, **第一句话**必先问取值范围, 不立即做 Phase 1:

```
请user拍取值范围(默认 = 本轮会话):
 A. 本轮(从本次会话最初对话开始 → 现在)= 默认
 B. 本日(0:00 - 现在, 跨会话)
 C. 本周(W{N} 周一 - 现在)
 D. 本月(M{N} 月 1 日 - 现在)
 E. 整个项目(从 your-project 仓库首次 commit / SOUL v0.1 创建至今)
 F. 自定义(user指定时间段 / 议题范围)

[等user拍 A / B / C / D / E / F 之一]
```

→ 不问 = 阿谀讨好型自动假设, 违反 SOUL § 十五.8 独立判断守则
→ 拍后再走 Phase 1-7
