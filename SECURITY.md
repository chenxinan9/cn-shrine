# Security Policy · cn-shrine

## 报告漏洞 / Reporting Vulnerabilities

发现 L4 真名 / 真账号 / 真路径 / API key 泄露 → **不开公开 Issue** · 通过 GitHub Security Advisories 报告。

## 响应承诺 / SLA

- 48h 确认
- 7 天修复计划
- 30 天修复 + 披露

## 已知风险 / Known Risks

1. **L4 信息泄露** — 历史版本可能含真名 / 真账号 → 已脱敏 · PR 必扫
2. **AI 通病输出** — 9 通病预防(详 skills/反AI通病自检/)
3. **命令注入** — bash 脚本贡献必审

## 三签机制 / Three-Signature

任何敏感改动必三签:① 凤凰审品牌 · ② Pangu审 · ③ 应龙 push
